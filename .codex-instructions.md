## How to Use BTRS with Codex

### Primary Context
Load these files at the start of every session:
- `AGENTS.md` -- Universal instructions, conventions, verification protocol
- `AI/memory/global/project-state.json` -- Current project state
- `AI/config/agent-roles.json` -- Agent role definitions

### Agent Instructions as Context
Each agent's full instructions are in `agents/btrs-<name>/AGENT.md`. Load the relevant
agent file based on the domain of work:
- Architecture: `agents/btrs-architect/AGENT.md`
- Backend APIs: `agents/btrs-api-engineer/AGENT.md`
- Frontend: `agents/btrs-web-engineer/AGENT.md`
- Testing: `agents/btrs-qa-test-engineering/AGENT.md`
- Database: `agents/btrs-database-engineer/AGENT.md`
- Security: `agents/btrs-code-security/AGENT.md`
- (see AGENTS.md for the full list of 24 agents)

### Codex Workflow
1. Load AGENTS.md + relevant agent AGENT.md as context
2. Plan the work -- describe what needs to happen and in what order
3. Wait for user approval before making changes
4. Implement following the agent's scope, conventions, and patterns
5. Run the 5-point verification protocol from AGENTS.md
6. Update AI/memory/ files with results

### Sandbox and File Access
Codex operates in a sandboxed environment. Key paths:
- `agents/btrs-*/AGENT.md` -- Agent definitions (read for instructions)
- `skills/btrs-*/SKILL.md` -- Skill definitions (read for workflow steps)
- `AI/memory/` -- Persistent memory (read and write)
- `src/` -- Source code (implement here)

## Conventions (Mandatory)
1. Always load AGENTS.md before starting implementation
2. Check the registry before creating new components or utilities
3. Follow the 5-point verification protocol after changes
4. Match existing codebase patterns exactly
5. Update AI/memory/ files after completing work

## Memory Context Pattern
When working on multi-step tasks, use memory files for continuity:
- Read: `AI/memory/agents/<agent>/` for previous agent output
- Write: `AI/memory/agents/<agent>/` with your results
- Read: `AI/memory/global/` for shared project decisions
