# BTRS Agents -- OpenAI Codex Instructions

See **AGENTS.md** for the full BTRS instruction set. This file adds Codex-specific patterns.

AGENTS.md is written in Codex-native markdown format. Load it directly as your primary instruction source.

## Setup

### For Codex CLI

```bash
# Clone BTRS
git clone https://github.com/btrs-ai/btrs.git ~/.btrs

# Copy AGENTS.md to your project as the instruction file
cp ~/.btrs/AGENTS.md ./AGENTS.md
```

Codex reads AGENTS.md automatically when present in the project root.

### For Codex in a specific project

```bash
git clone https://github.com/btrs-ai/btrs.git .btrs-agents
cp .btrs-agents/AGENTS.md ./AGENTS.md
```

## How to Use BTRS with Codex

### Primary Context
Load these files at the start of every session:
- `AGENTS.md` -- Universal instructions, conventions, verification protocol

### Agent Instructions as Context
Each agent's full instructions are in `agents/btrs-<name>/AGENT.md`. Load the relevant
agent file based on the domain of work:
- Architecture: `agents/btrs-architect/AGENT.md`
- Backend APIs: `agents/btrs-api-engineer/AGENT.md`
- Frontend: `agents/btrs-web-engineer/AGENT.md`
- Testing: `agents/btrs-qa-test-engineering/AGENT.md`
- Database: `agents/btrs-database-engineer/AGENT.md`
- Security: `agents/btrs-code-security/AGENT.md`

See AGENTS.md for the full list of 24 agents.

### Codex Workflow
1. Load AGENTS.md + relevant agent AGENT.md as context
2. Plan the work -- describe what needs to happen and in what order
3. Wait for user approval before making changes
4. Implement following the agent's scope, conventions, and patterns
5. Run the 5-point verification protocol from AGENTS.md
6. Update `.btrs/` vault files with results

## Conventions (from AGENTS.md)
1. Check `.btrs/conventions/registry.md` before creating new components or utilities
2. Follow the 5-point verification protocol after changes
3. Match existing codebase patterns exactly
4. Update `.btrs/` vault files after completing work (code-map, todos, changelog)
